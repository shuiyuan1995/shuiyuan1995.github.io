自己收集学习的一些面试题

其他

1. h5新增的标签有哪些？为什么要加强语义化？

1. html5增加了<main><header><footer><aside><section><video><audio><time><data>之类的语义化标签
2. 即使在没有CSS样式的条件下，也能很好地呈现出内容结构、代码结构；
3. 语义化HTML会使HTML结构变的清晰，有利于维护代码和添加样式；
4. 
2. 搜索引擎会利用爬虫分析抓取页面结构内容，有利于SEO，页面结构更加清晰，利于维护.提高页面可访问性，不会在样式加载不正常的时候一团乱，维持基本结构。利于部分残障人士页面阅读器解析

2. 为什么要清除浮动

1. 浮动脱离了正常文档流,会导致父元素高度塌陷，后面元素会挤到浮动元素下方，造成布局错乱。

3. 有哪几种存储数据方式，有什么不同？

1. cookie sessionStorage localStorage indexDb
2. cookie按设定的时间内有效，最多4k。
3. 两个storage都是html5新api,存储键值对,sessionStorage会话存储,页面访问时间内有效，关闭页面失效,localStorage一直有效，关闭页面不会失效，两个都是最大5M。
4. indexdb也是h5新api，浏览器内置小型数据库。

4. vue实现双向绑定的原理

1.  VUE实现双向数据绑定的原理就是利用了 Object.defineProperty() 这个方法重新定义了对象获取属性值(get)和设置属性值(set)的操作来实现的。
2. view到model则是一堆事件监听 输入框的input,选择组件的 change等等

5. 行内元素和块元素的区别？

1. 行元素无法设置宽高,行排布,不够换行
2. 块元素独占一行,可以设置宽高

6. 怎样实现文本不换行，超出部分省略？

7. vue生命周期有哪些？他们有神魔不同？

1. created,mounted,updated,destroyed 再加上四个before
2. created时data绑定，el不可见
3. beforemount调用render,实例添加$el替换el后调用mounted
4. 数据更新触发视图渲染则调用两个update
5. 销毁实例之前调用beforedestroy

8. 怎么实现跨域？jsonp原理是什么？

1. jsonp
2. 后端设置CORS
3. 各种代理 像nginx,node中间件
4. jsonp原理:利用script不受同源政策限制,将请求模拟成script标签请求,写入callback回调函数，后台将结果用callback包装返回，回调函数即可得到数据，不过jsonp只支持get请求.

9. es6有哪些新特性

1. const 与 let 变量
2. 模板字面量
3. 解构
4. for...of循环
5. 对象字面量简写法：使用和所分配的变量名称相同的名称初始化对象时如果属性名称和所分配的变量名称一样，那么就可以从对象属性中删掉这些重复的变量名称。
6. 展开运算符
7. 剩余参数(可变参数)
8. ES6箭头函数
9. 默认参数函数

10. canvas绘图，node.js，webpack了解

11. html里面，!doctype是做神魔的

1. 是一个声明，告诉浏览器用哪种HTML版本的规范来解读HTML文档。

12. css动画特效可以用js实现，为神魔还要用css

0. 让你的页面动画在移动设备上运行的更快一些：导致JavaScript效率低的两大原因：操作DOM和使用页面动画
1. 不占用JS主线程
2. 可以利用硬件加速
3. 浏览器可对动画做优化（元素不可见时不动画，减少对FPS的影响）

13. ajax拦截器

1. 通过 jQuery 提供的 ajaxSetup 方法，我们可以拦截页面上所有的 Ajax 请求响应（包括 .ajax、 .ajax、.ajax、.post、$.get）。

14. bootstrap中栏栅的24格是怎么做到的

1. bootstrap栅格可以分成24格，可以分别在12的基础上进行嵌套，每格嵌套一个，即可实现24格。

15. promise设计模式

1. Promise作为异步编程的一种解决方案。
2. Promise的出现改变了传统的异步编程方式，使JavaScript在进行异步编程时更加灵活，代码更加可维护、可阅读。

16. import和require的区别

1. 遵循规范：require 是 AMD规范引入方式，import是es6的一个语法标准，如果要兼容浏览器的话必须转化成es5的语法。
2. 调用时间：require是运行时调用，所以require理论上可以运用在代码的任何地方，import是编译时调用，所以必须放在文件开头。

17. git常用操作

1. 创建工作区，初始化工作区：git init
2. 添加新文件并添加到缓存区，并commit：git add readme.txt;git commit -m "create readme.txt";
3. 查看git 状态:git status;
4. 版本回退:git reset
5. 创建新的分支:git branch dev
6. 切换分支:git checkout dev
7. 合并得知 分支到当前分支:git merge dev
8. 删除分支:git branch -d dev

3. 常驻内存，会增大内存使用量，使用不当很容易造成内存泄露。

26. 前端常用的新能优化方法

1. 减少 HTTP请求数
2. 将脚本内容在页面信息内容加载后再加载
3. 懒加载
4. 将 CSS放在 HEAD中
5. 避免重复的资源请求
6. 减少不必要的 HTTP跳转
7. 避免使用 eval和 Function
8. DOM
9. with：with代码块的行为实际上是修改了代码块中的 执行环境 ，将obj放在了其作用域链的最前端，在 with代码块中访问非局部变量是都是先从 obj上开始查找，如果没有再依次按作用域链向上查找，因此使用 with相当于增加了作用域链长度。而每次查找作用域链都是要消耗时间的，过长的作用域链会导致查找性能下降。

5. 网络问题
6. 接收数据时间过长

28. vue中组件的data为什么是函数

1. 组件是可以被复用的
2. 如果是一个对象，当我们使用组件的时候当你修改一个属性的时候，data也会发生改变。
3. 当我们的data是一个函数的时候，每一个实例的data属性都是独立的，不会相互影响了。
